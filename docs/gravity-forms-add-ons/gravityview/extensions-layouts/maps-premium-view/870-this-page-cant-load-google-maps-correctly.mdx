# Error: This page can't load Google Maps correctly

If you experience this error message on your Maps View, this usually means there's a conflict between your active theme and GravityView or another active plugin and GravityView.

![Screenshot of the Google Maps error](https://s3.amazonaws.com/helpscout.net/docs/assets/54c58489e4b051242988532c/images/632e0f109c3e7c7d6a1ac1a2/file-A4dLo2RfU5.png)

In these situations, both plugins/themes are loading different Google API keys. To solve this problem, you will need to make sure both components are using the same API key.

###  Special instructions for some known plugins and themes

- [Divi](https://docs.gravitykit.com/article/869-adding-your-google-maps-api-key-into-divi)
- [Ultimate Addons for Elementor](https://docs.gravitykit.com/article/858-adding-your-google-maps-api-key-into-ultimate-addons-for-elementor)
- [Premium Addons for Elementor](https://docs.gravitykit.com/article/903-adding-your-google-maps-api-key-into-premium-addons-for-elementor)

### Avada

The setting is located under the Avada menu. Click on Theme Options, select Contact Form, and Google Map.

![Entering the Google Maps API key into the Theme Options page for Avada](https://s3.amazonaws.com/helpscout.net/docs/assets/54c58489e4b051242988532c/images/5d9b636e04286364bc9013fa/file-OwyP9g0qQ3.png)

### Enfold

In the sidebar, select Theme Options, then click on Google Services.

![Entering the Google Maps API key into the Theme Options page for the Enfold theme](https://s3.amazonaws.com/helpscout.net/docs/assets/54c58489e4b051242988532c/images/5d9b646604286364bc9013ff/file-polIMkXP1n.png)

###  Other known conflicting plugins/themes

- The Events Calendar
- Ubermenu
