# Disable the Custom Content code editor

GravityView uses the CodeMirror code editor on Custom Content fields and widgets. The code editor provides hints and auto-completing when writing code such as CSS or HTML.

![Screenshot of Custom Content field with code editor active](https://i.gravitykit.com/NpOXyo+)


:::info

If this solution is not good for you, please [contact us](https://www.gravitykit.com/support/) and let us know! We're gathering feedback on whether to add a GravityView-specific user setting to disable the code editor.

:::

  To disable the code editor in GravityView, follow these steps:

### 1. In the WordPress Dashboard, click on Users, then click on the submenu "Profile" to edit your profile settings.

![](https://i.gravitykit.com/C4dSA3+)

### 2. Check the box labeled "Syntax Highlighting" to disable syntax highlighting when editing code.

![](https://i.gravitykit.com/8xv0WP+)

### 3. Click the Update Profile button to save your setting.

![](https://i.gravitykit.com/m6gw6Z+)

#### 4. Return to your View and the code editor will not be used.

![Custom Content field with code editor inactive](https://i.gravitykit.com/fRRgwu+)
